# Fieldguide 界面与交互规格

> 版本：v0.4 | 状态：设计修订（Phase 1，VSCode/Obsidian 风格布局）  
> v0.4 变更：三栏布局 → 左文件树 + 右可分隔面板（2026-07-01）

---

## 一、设计目标

参考 **VSCode / Obsidian** 的开发者工作台体验：左侧项目文件树导航，右侧可自由分隔为 1~2 个面板，每个面板可在「图谱 / 代码阅读 / LLM 问答」间切换。

**代码地图核心能力**：由 **[Understand-Anything Dashboard](https://github.com/Egonex-AI/Understand-Anything)**（Phase 1 通过 iframe 嵌入）提供图谱交互、Tour、节点详情。Fieldguide 负责文件树、代码阅读器、LLM 问答面板、以及面板间联动。

详见 [understand-anything-integration.md](./understand-anything-integration.md) §4.3。

### 2.2 全局结构（v0.4 修订）

```
┌──────────────────────────────────────────────────────────────────────────┐

│            ├─────────────────────────┬───────────────────────────────────┤
│  📁 cmd/   │  面板 A（默认：图谱）      │  面板 B（可选）                    │
│   ├ main.go│  ┌───────────────────┐  │  ┌───────────────────────────────┐│

│            │  │  (iframe或代码阅   │  │  │                               ││

├────────────┴─────────────────────────┴───────────────────────────────────┤
│  状态栏：索引状态 · 节点数 · 当前项目名 · 面板布局指示                        │ ← 24px
└──────────────────────────────────────────────────────────────────────────┘
```

**布局规则**：
- **左侧文件树**：宽度 260px（默认），可拖拽调整 160px–400px，可一键折叠
- **右侧面板区**：flex-1，内部可拖拽分隔为 1 或 2 个面板（左右或上下排列）
- **面板 Tab**：每个面板顶部有 Tab 栏，可在「图谱」「代码」「问答」间切换
- **默认布局**：单面板显示图谱；用户可拖拽分出第二个面板
- **面板最小宽度**：280px

### 2.3 顶栏

- **Logo + 产品名**：点击回到项目库
- **主导航 Tab**：项目库 | 代码地图 | 理论 | 桥接（当前 Tab 下划线高亮）
- **面板布局控制**（代码地图 Tab 内）：`[ ⬜ ]` 单面板 / `[ ⬜｜⬜ ]` 左右双面板 / `[ ⬜ / ⬜ ]` 上下双面板 / `[ ⇄ ]` 交换左右面板
- **全局搜索（Ctrl+K）**：命令面板（重新索引、切换项目、打开设置）；代码语义搜索由 LLM 问答面板负责
- **设置**：API Key、语言、主题、关于

**规则**：未选中项目时，「代码地图 / 桥接」Tab 禁用并 tooltip「请先添加项目」。

### 2.4 首次启动引导

首次打开且 `onboardingCompleted !== true` 时，全屏 wizard 覆盖主界面。完整流程见 [onboarding-spec.md](./onboarding-spec.md)。

```
┌─────────────────────────────────────────────────────────────┐
│                    [半透明遮罩 #00000040]                      │
│         ┌─────────────────────────────────────┐             │
│         │  ● ○ ○ ○   Step 2 / 4               │             │
│         │                                     │             │
│         │  选择界面语言                         │             │
│         │  ○ 简体中文  ● 繁體中文  ○ English    │             │
│         │                                     │             │
│         │         [ 上一步 ]  [ 下一步 ]        │             │
│         └─────────────────────────────────────┘             │
│                      480px 居中卡片                           │
└─────────────────────────────────────────────────────────────┘
```

- 步骤：欢迎 → 语言 → 项目根目录 → 如何开始（Demo / 本地 / 稍后）
- 无关闭按钮；Esc 无效

## 三、视图规格

### 3.1 项目库

**空状态**（居中）：
- 插图 + 「添加你的第一个项目」
- 主按钮：「选择本地文件夹」
- 次按钮：「从 Git URL 克隆」

**列表项**（卡片或表格）：
- 项目名、主语言 badge、框架 tag
- 索引状态：就绪 / 索引中（进度环）/ 失败（可重试）/ **源码已变更**（橙色小圆点 + tooltip）
- 元信息：节点数、最后索引时间、`root_path` 摘要（hover 显示完整路径）
- 操作：打开地图 · 重新索引 · 删除

**添加 Git 对话框**：
- URL 输入、分支（可选）、浅克隆 checkbox
- **目标目录**：默认 `{projectsRoot}/{slug}/`
- 若未设置 `projectsRoot`：每次必须选择目录

---

### 3.2 代码地图（核心 — v0.4 重写）

> **实现来源**：图谱由 **UA Dashboard** iframe 嵌入提供；文件树、代码阅读、LLM 问答由 Fieldguide 自建。详见 [understand-anything-integration.md](./understand-anything-integration.md) §4.3。

#### 3.2.1 左侧：项目文件树

| 特性 | 说明 |
|------|------|
| **数据来源** | `scan-project.mjs` 扫描结果或直接读取项目目录 |
| **展示方式** | VSCode 风格树形目录，可折叠/展开 |
| **文件图标** | 按语言/文件类型显示图标（Lucide 图标 + FileIcon 组件，~15 种文件类型区分色） |
| **点击行为** | 在右侧「代码」面板打开文件 |
| **右键菜单** | 在资源管理器打开、复制路径 |
| **搜索过滤** | 文件树顶部内联搜索框（输入即过滤） |
| **索引状态叠加** | 已索引文件旁显示小绿点；未索引/新增文件灰点 |

**快捷键**：`Ctrl+Shift+E` 聚焦文件树。

#### 3.2.2 右侧：可分隔面板

**面板类型**（每个面板顶部 Tab 切换）：

| Tab | 名称 | 内容 | 实现来源 |
|-----|------|------|----------|
| 图谱 | Graph | UA Dashboard iframe 嵌入（交互式知识图谱） | UA Dashboard |
| 代码 | Code | Monaco Editor 或简单语法高亮文本区 | Fieldguide（只读预览） |
| 问答 | Chat | LLM 对话，上下文绑定当前项目/文件/节点 | Fieldguide |

**面板操作**：
- **添加面板**：点击面板区右上角 `[＋]` 或拖拽现有面板边缘分离
- **关闭面板**：面板 Tab 栏右侧 `[×]`，最后一个面板不可关闭
- **切换布局**：顶栏按钮切换左右/上下排列
- **拖拽分隔条**：调整两个面板的比例（50/50 默认，限制 20/80 ~ 80/20）
- **面板最大化**：双击面板 Tab 或 `Ctrl+Shift+M`，再次执行恢复

**默认布局**（打开项目时）：
- 右侧为**单面板**，默认显示「图谱」Tab
- 在文件树点击文件 → 仅更新活动面板的 filePath，不自动分屏（对齐 Obsidian 单文件打开行为）
- **双面板默认**：左 = 代码，右 = 图谱（可通过布局按钮交换）

**面板联动**（Phase 2）：
- 图谱中点击节点 → 代码面板跳转到对应文件:行号
- 代码面板选中符号 → 图谱高亮对应节点
- 问答面板引用节点 → 图谱/代码面板跟随

#### 3.2.3 图谱面板（UA Dashboard iframe）

以下能力 **由 UA Dashboard 提供**，Fieldguide 仅负责嵌入：

- 分层 / 力导向 / 业务域视图切换
- 缩放、平移、LOD（三级展开）、节点选中、邻居展开
- 节点摘要、源码预览（Dashboard 内嵌）、上下游关系
- Tour 控制条与步骤高亮
- Dashboard 内代码搜索

**Fieldguide 职责**：
- iframe 容器 + 加载状态
- 通过 `postMessage` 同步选中项目（加载对应 `knowledge-graph.json`）
- Phase 2：主题尽量统一

#### 3.2.4 代码面板（Fieldguide 自建）

| 特性 | Phase 1 | Phase 2+ |
|------|---------|----------|
| 文件内容展示 | 纯文本 + 简易语法高亮 | Monaco Editor 嵌入（只读） |
| 行号 | ✅ | ✅ |
| 符号大纲 | — | 文件内函数/类列表 |
| 「在 IDE 打开」按钮 | ✅ 右键菜单 | ✅ |
| 图谱联动 | — | 选中符号 ↔ 图谱高亮 |

#### 3.2.5 问答面板（Fieldguide 自建）

| 特性 | 说明 |
|------|------|
| **Phase 1** | 占位 — "问答功能 Phase 2 开放" |
| **Phase 2** | 接入 LLM，上下文绑定当前项目图谱 |
| **输入** | 底部输入框，支持 `@file` `@node` 引用 |
| **输出** | Markdown 渲染 + 代码块 + 可点击节点引用 |
| **上下文标签** | 顶部显示当前上下文：`📍 {project}` `📄 {file}` |

#### 3.2.6 索引进度

- 非阻塞：用户可浏览其他 Tab，状态栏显示进度
- 进度事件：`index:progress`（包装 UA pipeline 回调）
- 阶段文案 **以 UA 为准**，Fieldguide 做本地化映射：

| UA 阶段 | 状态栏文案 |
|---------|-----------|
| scan | `正在扫描文件… (1243 files)` |
| parse | `正在解析源码… (45%)` |
| analyze | `正在分析架构…` |
| build | `正在生成图谱…` |
| save | `索引完成 ✓` |

## 四、组件规范

### 4.1 文件树

- 使用虚拟滚动（大量文件时不卡）
- 目录箭头：▸ 折叠 / ▾ 展开
- 缩进：每层 16px
- 选中项高亮：VSCode 风格（`#E4E6F1` 浅色 / `#2A2D3E` 深色）
- 字体：`Segoe UI` 13px

### 4.2 可分隔面板

- 分隔条：4px 宽，hover 时变色（`#3B82F6`），cursor: col-resize / row-resize
- 面板 Tab 栏：高度 32px，Tab 宽度自适应，活跃 Tab 底部蓝色下划线
- 面板最小宽度：280px

### 4.3 Tab 切换动效

| 场景 | 动效 | 时长 |
|------|------|------|
| 面板 Tab 切换 | 内容区淡入 | 100ms |
| 面板添加/关闭 | 宽度过渡 | 200ms |

### 4.4 ThoughtChain（思考链）

LLM 问答面板中，每步 ReAct 可折叠：

```
▸ Thought    我需要先找到 HTTP 入口...
▸ Action     search_code({ query: "main listen" })
▸ Observation  找到 gateway/main.go ...
▸ Answer     （最终回答区域）
```

### 4.5 命令面板（Ctrl+K）

Fieldguide 壳层命令（Phase 1）：
- 切换项目、重新索引、打开设置、切换主题
- 跳转文件（打开代码面板）

## 五、视觉规范

### 5.1 主题

**主题预设**（5 套可选，默认 `parchment`）

| 预设 | 类型 | 说明 |
|------|------|------|
| parchment (默认) | 浅色 | 米白纯色底 `#FDFCF8`，白卡片，鼠尾草绿强调 `#4A8B71`，现代笔记本质感 |
| forest | 深色 | 深绿底色 `#1B2E1E`，草木绿强调 `#7DBF6E` |
| slate | 浅色 | 蓝灰底 `#F0F2F5`，紫罗兰强调 `#6366F1` |
| midnight | 深色 | GitHub Dark 风格 `#0D1117`，天蓝强调 `#79C0FF` |
| paper-dark | 深色 | 暖暗底 `#2D2420`，金棕强调 `#D4A76A` |
| none | — | 无预设，跟随系统亮/暗默认值 |

**parchment 默认色值**（v2 · 对齐 sampleTheme.png）：

| 令牌 | 值 |
|------|-----|
| `--fg-bg` | `#FDFCF8` |
| `--fg-card` | `#FFFFFF` |
| `--fg-border` | `#E0E0E0` |
| `--fg-text-primary` | `#222222` |
| `--fg-text-secondary` | `#757575` |
| `--fg-accent` | `#4A8B71` |
| `--fg-accent-muted` | `#EBF4F0` |
| `--fg-sidebar-bg` | `#F8F7F4` |

**parchment 背景**：纯色米白 `#FDFCF8`（无网格纹理）；侧栏 `#F8F7F4`。

**主题系统**：所有颜色通过 CSS 语义令牌 `--fg-*` 定义（`tokens.css`），组件禁止使用硬编码 Tailwind 颜色类。覆盖顺序：`:root` 默认 → `[data-theme="dark"]` → `[data-theme-preset="..."]`。

**浅色（none 预设 / 无预设）**：
- 背景 `#FAFAFA`，卡片 `#FFFFFF`，边框 `#E5E7EB`
- 文字 primary `#111827` / secondary `#6B7280`
- 文件树选中 `#E4E6F1`

**深色**：
- 背景 `#0F172A`，卡片 `#1E293B`，边框 `#334155`
- 文字 primary `#F8FAFC` / secondary `#94A3B8`
- 文件树选中 `#2A2D3E`

### 5.2 字体

- UI：`Segoe UI`, system-ui, sans-serif
- 代码：`Cascadia Code`, `Consolas`, monospace
- 字号：正文 14px，标题 16–20px，代码 13px，文件树 13px

### 5.3 图谱节点（UA Dashboard 负责）

| 类型 | 说明 |
|------|------|
| file | 圆角矩形 |
| function | 椭圆 |
| class | 六边形 |
